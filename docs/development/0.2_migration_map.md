# OpenPFC 0.1 → 0.2 migration map

**Live document.** Every milestone (M1–M12) that adds, merges, or deletes a
public concept updates this table. It is the single index of what replaces what
and when the old thing is removed. Seeded from the architecture audit §14
(`OPENPFC_ARCHITECTURE_AUDIT.md`) and the execution plan
(`OPENPFC_REFACTORING_EXECUTION_PLAN.md`).

Status legend: **planned** · **in progress** · **done**.

## Temporary migration adapters (must not survive 0.2.0)

| ID | Adapter | Introduced | Removed | Parity test |
|----|---------|-----------|---------|-------------|
| A0 | `pfc::World` as a deprecated alias/wrapper over `Domain` | M1 | M12 | golden trajectories unchanged via A0 |
| A1 | `pfc::compat::LegacyModelPhysics` (wrap Gen-1 `Model` as a physics concept) | M7 | M12 | diffusion fixture: Gen-1 `Simulator` vs A1+new driver, bitwise CPU |
| A2 | `Simulator::step_with_physics` bridge | M7 (adopt) | M12 | its dedicated test added in M7 |

No newly written functionality may depend on A0–A2.

## Enforcement added in M0 (done)

- Layering check extended to `kernel !→ frontend`, `kernel !→ runtime`,
  `runtime !→ frontend` — `scripts/check_kernel_no_frontend_includes.sh`. The
  checker carries a `--self-test` fixture (clean tree passes, a deliberate
  violation is detected); both the real check and the self-test are required
  CI steps in the `code-quality` job.
- `OpenPFC_CTEST_DISCOVER=ON` shards non-MPI CTest per Catch2 case in CI; a CI
  step asserts the shard count equals the binary's `~[MPI]` enumeration (no case
  lost versus the monolithic run) and records the sharded-run wall-time.
- ADRs 0004–0008 accepted.
